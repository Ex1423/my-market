# 核心交互协议：先对齐意图，再执行代码

1. **禁止直接输出代码**：当我提出新的功能需求、修改建议或逻辑重构时，你严禁直接修改或生成代码。
2. **第一步：理解与反馈**：你必须先用中文复述你对我的意图的理解，并列出：
   - 核心目标：你认为我要解决的核心问题是什么？
   - 技术方案：你准备采用什么逻辑或 API？
   - 潜在风险：是否存在逻辑矛盾、性能隐患或不清晰的地方？
3. **第二步：等待授权**：在回答的最后，必须询问：“我的理解是否准确？如果没问题，请回复‘开始’，我将为您生成代码。”
4. **第三步：正式执行**：只有当我回复“开始”、“好”、“Go”或类似确认指令后，你才能正式编写代码。

# 个人偏好
- 使用中文沟通。
- 代码注释要清晰。
- 如果是 SketchUp 开发，必须遵守 Ruby API 规范。
